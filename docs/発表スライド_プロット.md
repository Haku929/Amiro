## 1. オープニング・テーマ（1分）

### スライド1: タイトル
- **Amiro（アミロ）**
- 名前の由来: **Amo**re（愛する）＋ **mir**ror（鏡）＋ **iro**（色）＝ 鏡に写った自分の色を愛する
- サブタイトル: 「理想の相手」ではなく、「好きな自分でいられる相手」と出会う

### スライド2: ハッカソンのテーマ → 私たちの解釈
- テーマ（例: 「つながり」「マッチング」など、実際のテーマに差し替え）
- **私たちの着目点**: 従来のマッチングは「相手の条件に合わせる自己犠牲」。マッチしても「演じ続ける自分」で疲れてしまう
- **問い**: 本当に大事なのは「誰といる時の自分が一番好きか」では？

---

## 2. 作ったものの説明（2分30秒）

### スライド3: コンセプト
- **鏡に写った自分の色を愛する**  
  AIや相手は「鏡」。その鏡に映った、いつもと違う自分（色）に気づき、好きになれる相手と出会う
- 人は相手によって違う自分が立ち上がる。Amiroは、その「好きな色」を軸にマッチングする

### スライド4: サービスの流れ（3ステップ）
1. **AIという鏡での自己発見**  
   毎日3体の鏡と対話 → 鏡に映った自分の「イロ」をBig5で可視化
2. **好きな色をストック**  
   気に入った自分のイロを最大3つまでスロットに保存
3. **相互共鳴でマッチング**  
   自分の鏡に映る「理想のイロ」と相手の「イロ」が響き合う相手をベクトル検索で発見

### スライド5: 技術スタック（1枚で簡潔に）
- Next.js / Supabase（pgvector） / Gemini 2.5 Flash / Recharts
- 「AI・心理学・ベクトル検索の融合」と一言で伝える

### スライド6: 画面構成の概要
- ホーム（AIガチャ）→ チャット → レポート → プロフィール → 共鳴マッチング → 共鳴詳細 → 1対1DM
- 図やスクショで1枚にまとめる

## 3. 実演（3分30秒）

**事前準備**: デモ用アカウント・保存済みスロット・マッチング相手を用意。API障害時のフォールバック（モック・キャッシュ）を確認。

### 実演フロー（ストーリーで繋ぐ）
1. **ホーム**（30秒）  
   「今日の3体の鏡」を見せ、1体を選択
2. **AIチャット**（1分）  
   数往復会話 → 「終了して分析」まで
3. **レポート**（45秒）  
   Big5グラフと要約 → 「この自分を愛する」でスロット保存
4. **共鳴マッチング**（45秒）  
   スコアの高い相手一覧 → 1人選択
5. **共鳴詳細**（30秒）  
   二人の関係性解説・ベクトル比較 → 「チャットする」でDMへ

**ポイント**: 説明は最小限に。操作と画面の動きで伝える。止まったら30秒で切り上げる想定で。

## 5. 苦労した点・良かった点・改善点（2分）

### スライド8: 苦労した点
- AI分析の精度安定（10往復だけでBig5を出す難しさ）
- 5人並列開発でのマージ・結合の調整
- デモ時のAPI障害・レート制限への備え

### スライド9: 良かった点
- 「AIと会話してユーザーの性格を測定したい！」というアイデアをチームメンバーでどんどん発展させて面白い企画ができたと思う
- Next.js + Supabase + Gemini の組み合わせで開発速度が出た
- 仕様書・APIスキーマを先に固めたことで役割分担が明確だった

### スライド10: 改善点・今後
- 会話履歴の永続化・分析プロンプトのチューニング
- 共鳴詳細のAI解説の精度向上
- スケール時のマッチングAPI最適化（pgvector近似検索など）
